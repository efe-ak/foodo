Assets Structure

JavaScript

All the static JavaScript files are located in dist/<theme_name>/js.

Application scripts

Standalone JavaScript Modules

As a convenience, we provide the entire UI framework broken down in separate modules. Some of the standalone modules were not used with every theme but ALL of the modules are 100% compatible with any of the provided themes.

  • module-chat.js - the required JavaScript for Chat components
  • module-essentials.js - the required JavaScript for Essential UI components such as Tabs and Tree Views
  • module-layout.js - the required JavaScript for Layout components
  • module-maps.js - the required JavaScript for Maps components (Vector Maps & Google Maps)
  • module-media.js - the required JavaScript for Media components, such as Carousels, Galleries and Media Players
  • module-navbar.js - the required JavaScript for Navbar components
  • module-sidebar.js - the required JavaScript for Sidebar components
  • module-timeline.js - the required JavaScript for social Timeline components

All Standalone JavaScript Modules Bundle

Each theme includes an extra Standalone JavaScript Modules Bundle that contains ALL THE AVAILABLE standalone JavaScript modules. The bundle file name is module-bundle-all.js. There is also a minified version module-bundle-all.min.js within the same directory.

Theme-specific Standalone JavaScript Modules Bundle

Each theme includes a Theme-specific Standalone JavaScript Modules Bundle that contains only the JavaScript modules used for the theme as showcased in the demo. The bundle file name is module-bundle-main.js. There is also a minified version module-bundle-main.min.js within the same directory. This is the default file loaded in every theme and it's the recommended way, even more so if you're using the grunt watchers included with the development suite.

All Vendor Scripts Bundle

Each theme includes a Vendor Scripts Bundle with ALL THE AVAILABLE 3rd party JavaScript libraries. The vendor bundle file name is vendor-bundle-all.js. There is also a minified version vendor-bundle-all.min.js within the same directory.

  • Includes, in this exact order:

    1. vendor-core.js
    2. vendor-tables.js
    3. vendor-forms.js
    4. vendor-media.js
    5. vendor-charts.js
    6. vendor-maps.js
    7. vendor-tree.js
PRO TIP: You can get the full list of all the various libraries installed and the versions used and also check for available updates for each library in seconds, anytime! Just go into a Terminal to the dev directory and type bower list. After updating a package using Bower, you only need to run grunt --theme your_theme_name build:dist to recreate all the bundles with the updated plugins.

Styling

This amazing theme supports CSS pre-processing, not just from one, but two of the top industry-approved and mass-adopted CSS pre-processors: Less and Sass (CSS with superpowers) .

All the styling source is located within dev/app/less or dev/app/scss directories and are broken down in the following:

<common> <!-- Common Styling used for all Themes -->
├── <components>        <!-- Common components used throughout multiple themes -->
├── <layout>            <!-- Holds the common styling for the content layout, navbars, sidebars and footer -->
├── <mixins>            <!-- Common Mixins -->
├── <variables>         <!-- Common variables used throughout multiple themes -->
├── _helpers.less       <!-- Common Styling Helpers -->
├── _scaffolding.less   <!-- Common Scaffolding -->
├── _demo.less          <!-- A partial loader for styling used only for demo purposes -->
├── _vendor.less        <!-- Loads the main Vendor styles (Bootstrap, Font Awesome, Bootstrap Select) -->
├── common.less         <!-- The main common partial loader -->
│
<demo> <!-- Styling used for DEMO purposes (can be removed) -->
├── skins.less          <!-- Styling used for the Skin Switcher -->
├── demo.less           <!-- Other -->
│
<skins> <!-- Styling used for Skins -->
<themes> <!-- Specific Theme Styling -->
├── <1>
│   ├── <layout>        <!-- Loads the common layout styles and holds custom theme layout styling,
│   │                        layout fixes, navbars and sidebars -->
│   │
│   ├── _variables.less <!-- Custom Theme Variables -->
│   └── default.less    <!-- Loads the required partials for theme 1 -->
│
